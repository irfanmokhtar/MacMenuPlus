MacMenu+

A macOS menu-bar utility — clipboard history, window switcher, and window tiling in one panel. No Dock icon.

Everything runs from a single menu-bar panel plus global hotkeys. Three features share that panel, stacked top to bottom: Clipboard, Open Windows, and Tile Window.

📋 Clipboard History

Automatically captures whatever you copy and keeps a searchable, scrollable history.

  • Text & images — captured automatically (the pasteboard is polled every 0.5 s).
  • Search — filter the history by typing in the search field.
  • Pin / unpin — pinned items survive a "clear" and are never evicted by the capacity limit.
  • Configurable capacity — 10–200 items (default 50), set in Settings. Oldest unpinned items drop off first.
  • Click to copy back — selecting an item puts it back on the clipboard (without creating a duplicate entry).
  • Clear — wipe all unpinned items in one click.

History is in-memory only — it is cleared when you quit. There is no on-disk persistence.

🪟 Window Switcher

Lists every open window across all apps — not just one per app — and lets you jump straight to any of them. Available two ways:

In the panel — "Open windows"

  • A scrollable list of windows; click any row to raise that window.
  • Resizable — drag the handle below the list to grow/shrink it; the height is remembered.

HUD overlay (hotkey)

  • Press ⌃⌥Tab for a floating switcher overlay, centered on screen.
  • Fully keyboard-driven: Tab / ⇧Tab and arrow keys to move, Return to switch, Esc to cancel.
  • A single Tab jumps to the next window — classic ⌘Tab behavior.

Includes minimized and hidden windows, and correctly handles Chromium/Electron apps (Chrome, Slack, VS Code) that expose their windows lazily.

Raising a specific window (vs. just focusing the app) requires Accessibility permission. Without it, the switcher can still bring the app forward.

🔲 Window Tiling New

Snap the active window into halves, quarters, maximize, or center — including the quarter positions macOS has no built-in shortcut for.

From the panel — "Tile window"

A grid of buttons applies a tile to the last app you were using (opening the panel takes focus, so tiling targets the previously-active window). The header shows which app will be affected.

By hotkey

Tiling hotkeys act on the current frontmost window. Defaults follow the popular Rectangle layout:

Left half⌃⌥←
Right half⌃⌥→
Top half⌃⌥↑
Bottom half⌃⌥↓
Top-left⌃⌥U
Top-right⌃⌥I
Bottom-left⌃⌥J
Bottom-right⌃⌥K
Maximize⌃⌥↩
Center⌃⌥C

Multi-monitor aware (tiles within the display the window is on) and respects the menu bar / Dock. Every shortcut is editable in Settings.

Tiling moves & resizes windows via the Accessibility API, so it needs Accessibility permission to work.

🔐 Permissions & Requirements

  • macOS 14+. Xcode 15+ to build from source.
  • Accessibility permission powers window raising and tiling. Grant it at System Settings → Privacy & Security → Accessibility → MacMenu+. The app prompts on first use of a feature that needs it.
  • Not sandboxed — the Accessibility API is incompatible with App Sandbox.
  • No Dock icon — runs as a menu-bar accessory (LSUIElement).
MacMenu+ · SwiftUI + AppKit · single dependency: KeyboardShortcuts. See README.md for build instructions.
